Плюсы:
- Наличие резервирование, даже при условии выхода одного из облачных провайдеров
- Близость к пользователю за счет GEODNS
- Масштабируемость (Легко добавить/удалить нового облачного провайдера без влияния на сервис)

Минусы:
- Стоимость решения
- Сложность решения 

Особенности:
- Cloudflare использовать не обязательно, можно развернуть свои балансировщики (по типу Citrix Netscaler) с GSLB, по одному в каждом облаке и балансировать DNS самостоятельно.
- При наличии собственных PI сетей, есть возможность уйти на BGP.
